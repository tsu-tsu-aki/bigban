# ニュース運用マニュアル (microCMS)

> 対象: 非エンジニアの編集者・運用担当者
> 最終更新: 2026-04-25

サイトの「ニュース」(`/news`, `/news/[slug]`, About 内 05 NEWS) は **microCMS** で管理されています。本書は記事の投稿・編集・公開・トラブル対応の手順書です。

---

## ログイン

1. ブラウザで [https://app.microcms.io/](https://app.microcms.io/) を開く
2. 招待メールで設定したメールアドレス + パスワードでログイン
3. `THE PICKLE BANG THEORY` サービスを選択
4. 左メニューから **「news」** API を選択

---

## 新規ニュースの公開

### 1. 「新規追加」をクリック

### 2. フィールドを入力

| フィールド | 必須 | 入力例・注意 |
|---|---|---|
| **タイトル** | はい | 「グランドオープン記念 6/1 (土) イベント開催」 |
| **スラッグ** | はい | URL の末尾になる半角英数+ハイフン (例: `grand-opening-2026`)。**ロケール内ユニーク**。一度公開した記事のスラッグは変更しない (SEO 影響大) |
| **言語** | はい | `ja` または `en` を 1 つだけ選択。両言語で出すなら同じスラッグで 2 レコード作成 |
| **カテゴリ** | はい | `notice`/`media`/`event`/`campaign` から 1 つ以上選択 |
| **抜粋** | はい | 160 字以内。description / OGP / JSON-LD に使われる。AI 生成プロンプトでも一括生成 |
| **表示モード** | はい | `html` (AI 生成 HTML) / `rich` (リッチエディタ) のどちらかを必ず選択 |
| **本文 (HTML)** | 任意 | `表示モード=html` の時に貼り付け。AI 生成 HTML はここへ |
| **本文 (リッチ)** | 任意 | `表示モード=rich` の時に手書き。簡単な記事はこちら |
| **アイキャッチ** | 任意 | 推奨 16:9、1200px 幅以上。未設定でもカテゴリ色のグラデーションで自動描画 |
| **外部リンク** | 任意 | `{label, url}` ペア。`url` は `https://` 必須 (`http://` や `javascript:` は弾かれる) |
| **公開日時 (publishedAt)** | 任意 | 標準フィールド。予約投稿に使う (下記参照) |

### 3. 右上「公開」

公開後 1〜2 秒で本番サイトに反映 (Webhook 経由 ISR)。

---

## 下書き & プレビュー

1. 「公開せず保存」または公開後の「内容を編集」で **下書き** 状態
2. 右上の「画面確認」→「下書きで確認」ボタンを開く
3. 下書きプレビュー URL は次の形式で構築されます:

```
https://www.thepicklebang.com/api/draft/enable?secret=${MICROCMS_DRAFT_SECRET}&contentId={CONTENT_ID}&draftKey={DRAFT_KEY}
```

(microCMS 管理画面の「画面プレビュー設定」で上記テンプレートを登録済み)

> **注**: microCMS が置換するのは `{CONTENT_ID}` と `{DRAFT_KEY}` のみ。サイト側で contentId から slug/locale を逆引きしてリダイレクトします。

4. プレビュー画面上部に **黄色いバナー**「プレビューモード中」が表示されます
5. プレビュー終了は同バナーの「終了」をクリック

> プレビュー URL は **secret 必須**で、社外公開しないでください。

---

## 予約投稿

1. 編集画面で「公開予約」を選び、`publishedAt` を未来日時に設定
2. 「予約」をクリック
3. 設定時刻に microCMS が自動で公開 → サイトに反映 (Webhook ISR)

> 予約投稿の最小単位は分。秒指定不可。

---

## 日英両対応の記事

- 同じスラッグで **2 レコード** (ja / en) を作成
- 詳細ページ右上の言語切替ボタン (EN / JA) で切替可能
- 片言語のみで運用する場合: 該当言語のレコードのみ作成
  - 言語切替ボタンは押せるが、対向言語の **一覧ページ** にフォールバック
- 翻訳タイミングが揃わない場合: 先に ja を公開 → 後日 en を追加で OK

## カテゴリの使い分け

| カテゴリ | 用途 |
|---|---|
| `notice` (お知らせ) | 営業時間変更、休館、システムメンテナンスなど |
| `media` (メディア掲載) | 雑誌・Web 媒体に取り上げられた告知 |
| `event` (イベント) | 来場型イベント、キャンペーン期間中のレッスン会など |
| `campaign` (キャンペーン) | 期間限定の入会キャンペーン、紹介特典など |

複数該当する場合は複数選択可。

## 画像

- microCMS のメディア管理にアップロード → 自動で `images.microcms-assets.io` ホストで配信
- アイキャッチは **16:9 推奨**、1200px 幅以上で
- 本文中の画像も自動で width/height/loading=lazy が補完されます (ない場合は警告ログ)
- 外部画像 (例: pixabay, unsplash 直リンク) は **本文中で使えません** (サニタイザが弾く)

## 拡張 HTML 要素 (表 / バッジ / 注記ボックス)

イベント告知などで使える要素を許可しています。HTML モード (`displayMode=html`) で `bodyHtml` フィールドに直接記述してください。

### 表 (table)

料金表・キャンセル規定・カテゴリ比較などに使用。

```html
<table>
  <caption>キャンセル規定</caption>
  <thead>
    <tr>
      <th scope="col">キャンセル日</th>
      <th scope="col">キャンセル料</th>
    </tr>
  </thead>
  <tbody>
    <tr><td>15日前まで</td><td>無料</td></tr>
    <tr><td>14〜2日前</td><td>50%</td></tr>
    <tr><td>前日・当日</td><td>100%</td></tr>
  </tbody>
</table>
```

**重要**:
- 1 行目は必ず `<th scope="col">` (列見出し) を使う
- `<caption>` で表の用途を 1 文で記述すると SEO/a11y が向上
- モバイルでは自動で横スクロール可能になる (キーボード矢印キー対応)

### バッジ (badge)

「中級」「重要」など、インラインで小さなラベルを付けたい時:

```html
<p>レベル: <span class="badge">中級</span> <span class="badge">DUPR 3.5+</span></p>
```

**重要**: 色だけで意味を伝えない (例: 「赤=危険」のような区別はしない)。テキストで意味を明示する。

### 強調マーカー (mark / highlight)

本文中の重要語を蛍光ペン風にハイライト:

```html
<p><mark>最低3試合保証</mark>、ペアがいない方の単独エントリーも歓迎します。</p>
```

または `<span class="highlight">` でも同じ表示。

### 注記ボックス (note / caution)

補足説明や注意喚起:

```html
<aside class="note">
  <p>注意: 駐車場はございません。公共交通機関でお越しください。</p>
</aside>

<aside class="caution">
  <p>申込期限を過ぎたキャンセルは100%のキャンセル料が発生します。</p>
</aside>
```

`note` は通常の補足、`caution` はより注意喚起レベル (上部ラベルが NOTE / CAUTION で切り替わる)。

**重要**: 緊急通知 (`role="alert"`) ではなく、編集上の補足です。`<p>` で囲んで本文を入れてください。

### 使用してはいけないタグ

- `<h1>`: 記事タイトルは page 側で出力するため、本文では h2 以下のみ使用
- `<script>`, `<iframe>`, `<style>`, `<form>`, `<input>`: セキュリティ上の理由で除去
- `style="..."` 属性, `onclick="..."` 等のイベント属性: 同上
- 任意の class 名: `lead`, `caption`, `badge`, `highlight`, `note`, `caution` 以外は除去される

## トラブルシューティング

| 症状 | 原因と対処 |
|---|---|
| 公開しても本番に反映されない | Webhook エラーの可能性。microCMS 管理画面の「Webhook 履歴」で 200 が返っているか確認 |
| プレビューが 401 | `MICROCMS_DRAFT_SECRET` の不一致、または `slug` の形式違反 (大文字を含むなど)。slug は `^[a-z0-9-]+$` のみ |
| 詳細ページが 404 | スラッグタイプミス、または該当言語のレコードが未作成 |
| 本文の特定タグや属性が消えた | サニタイザの仕様 (script / iframe / style / form / on* 属性 等は除去) |
| 画像が表示されない | 外部ホストの画像は弾かれる。microCMS にアップロードしてから本文に挿入する |
| 「本文がありません」と出る | `bodyHtml` と `body` が両方空の状態。表示モードに応じて埋める |
