# THE PICKLE BANG THEORY — ホームLP デザイン仕様書

## 概要

ホームページ（`src/app/page.tsx`）を、9つのコンテンツセクション＋固定ヘッダー＋フッターで構成されるシングルページLP（ランディングページ）にリデザインする。すべてのセクションは縦に連なり、ナビゲーションリンクはページ内の各セクションへスムーススクロールで移動する。

デザインはプレミアムスポーツブランド（Nike、On Running）を参考にした、エディトリアル（雑誌編集的）クオリティの美学に従う。ダークなシネマティックトーン、非対称レイアウト、セリフ体×サンセリフ体のコントラストによる特大タイポグラフィ、スクロール連動アニメーション、意図的な余白を特徴とする。

## アーキテクチャ

### 新規作成ファイル

```
src/components/home/
  HomeNavigation.tsx      — 固定ヘッダー（スクロール変化、JP/ENトグル、RESERVEボタン）
  HomeHero.tsx            — フルビューポートヒーロー（非対称レイアウト、テキストリビール）
  HomeConcept.tsx         — オフホワイトセクション（縦書きラベル、段落フェードイン、パララックス写真）
  HomeKeyNumbers.tsx      — ダークセクション（カウントアップ数字、セリフ体タイポグラフィ）
  HomeFacility.tsx        — ダークセクション（コートショーケース＋アメニティリスト）
  HomeServices.tsx        — 5つのフルワイドサービス行（背景色交互切り替え）
  HomePricing.tsx         — ダークセクション（近日公開＋3カラムプレビュー＋レンタル案内）
  HomeFounder.tsx         — シネマティック写真背景（タイムライン経歴、プレスリンク）
  HomeAccess.tsx          — オフホワイトセクション（Google Maps iframe、2カラム住所・アクセス）
  HomeContact.tsx         — ダークセクション（お問い合わせフォーム（Resend）＋連絡先情報）
  HomeFooter.tsx          — ダークフッター（アクセントセパレーター、3カラム）

src/hooks/
  useLanguage.ts          — JP/EN 状態管理（React context + state）
  useActiveSection.ts     — IntersectionObserverによるナビハイライト
  useMagneticButton.ts    — マグネティックホバー効果

src/app/api/contact/
  route.ts                — POSTエンドポイント（入力バリデーション、Resend経由メール送信）
```

### 変更なしのファイル

既存コンポーネント（`Hero.tsx`、`Introduction.tsx`、`KeyNumbers.tsx`、`Concept.tsx`、`Services.tsx`、`Founder.tsx`、`Footer.tsx`、`Navigation.tsx`）はそのまま残す。`/facility` や `/services` ページで引き続き使用される可能性があるため。

### 変更するファイル

- `src/app/page.tsx` — 現在のコンポーネント構成を新しい Home* コンポーネントに置き換え
- `src/app/globals.css` — 必要に応じて新規ユーティリティクラスを追加（ヒーロー用カスタムカーソル等）

## ロゴ使用ルール

既存のロゴアセット（`public/logos/`）を使用する。ネオン版はteaserページ専用のため、ホームLPでは白版を使用。

| 場所 | ファイル | 理由 |
|---|---|---|
| ヘッダー（デスクトップ） | `yoko-w.png`（横型・白） | ナビバーに自然に収まる横長比率 |
| ヘッダー（モバイル） | `mark-w.png`（マーク単体・白） | 省スペース |
| フッター | `yoko-w.png`（横型・白） | ヘッダーと統一 |
| オフホワイト背景セクション | テキスト表記のみ | 画像ロゴはダーク背景用のため |

## タイポグラフィ

- **見出し（セリフ体）：** DM Serif Display（読み込み済み）。ヒーロー：デスクトップ80〜120px、モバイル36〜48px
- **本文・UI（サンセリフ体）：** Inter + Noto Sans JP（読み込み済み）。本文：16〜18px
- **ブランド名：** 常に大文字、字間広め、サンセリフ体
- **スケールコントラスト：** 大きな見出し vs 小さな本文がメインのデザインツール

## セクション仕様

### 固定ヘッダー — HomeNavigation

- 配置：fixed、全幅、z-50
- ヒーロー上：透明背景
- スクロール時：backdrop-blur＋半透明ダーク背景（`#0A0A0A/80`）
- **デスクトップレイアウト：**
  - 左：ロゴ「THE PICKLE BANG THEORY」（大文字、字間広め、サンセリフ体）
  - 中央：ナビリンク（CONCEPT / FACILITY / SERVICES / PRICING / ACCESS / CONTACT）— 小さな大文字、ホバー時アンダーライン拡張アニメーション、現在表示セクションをアクセントカラーでハイライト
  - 右：JP/ENトグル（小テキスト、選択中=白、非選択=グレー、`/`で区切り）＋ RESERVEボタン（アクセント背景、黒テキスト、小さめ、角丸なし）
- **モバイルレイアウト：**
  - 左：ロゴ（短縮版または小さめ）
  - 右：JP/ENトグル＋ハンバーガーアイコン
  - ハンバーガーを開くとフルスクリーンのダークオーバーレイ。大きな縦積みテキストでセクション名を表示。タップでスクロール＋自動クローズ。下部にRESERVEボタン
- アクティブセクション検知：`useActiveSection` フックによるIntersectionObserver

### §1 ヒーロー — HomeHero

- フルビューポート（100vh）、ダーク背景
- **デスクトップ：** 非対称分割レイアウト
  - 左側 約60%：特大セリフ体の見出しを縦に積む：
    「ここから、ピックルボールのビッグバンが始まる。」
    — 各行がスタガードアニメーションで順次表示
    — 「ビッグバン」をアクセントカラー（#C8FF00）でハイライト
  - 右側 約40%：フルハイトの雰囲気ある写真プレースホルダー（スイング中の選手、ドラマチックなサイドライティング、浅い被写界深度、暗い背景）
    — 斜めカットまたは角丸でマスク（単純な四角形にしない）
    — スクロール時の微かなパララックス
  - 見出しの下に小さな英語タグライン（字間広めの大文字サンセリフ体）："FROM A SMALL DINK TO A BIG MOVEMENT"
  - CTAボタン1つ：「RESERVE A COURT →」— アクセント背景、黒テキスト、角丸なし、`useMagneticButton`によるマグネティックホバー
  - 最下部：スクロールインジケーター — ミニマルなアニメーション矢印、または縦に回転する「SCROLL」テキスト
- **モバイル：** 縦積み。見出しを上（36〜48px）、写真を下（またはバックグラウンドとして薄く）。CTAボタンはフルワイド
- ヒーローセクションでカスタムカーソル効果（デスクトップのみ、タッチ/モバイルでは無効）

### §2 コンセプト — HomeConcept

- 背景：オフホワイト（#F5F2EE）、テキスト：#1A1A1A
- **デスクトップ：**

## 言語切り替え（JP/EN）

- React context（`useLanguage` フック）が `language` ステートと `toggleLanguage` 関数を提供
- `LanguageProvider` がページ全体をラップ
- 主要なテキストのみ翻訳対応（見出し、ナビラベル、タグライン、ボタンラベル、フォームラベル）
- 本文の詳細な説明文は元の言語のまま（日本語の説明文はそのまま、英語のブランド要素もそのまま）
- 翻訳データは各コンポーネント内にシンプルなオブジェクトとして格納（外部i18nライブラリは不使用）

## アニメーション・モーション

| 要素 | アニメーション | デスクトップ | モバイル |

| ヒーロー見出し | 行ごとのリビール（overflow hidden + y-transform） | あり | あり（簡略化） |
| スクロールセクション | 軽い上方向の動きと共にフェードイン（順次） | あり | あり |
| 写真 | useScroll + useTransformによる微かなパララックス | あり | なし（静的） |
| ナビゲーション | 透明→blur＋ダークにスクロール変化 | あり | あり |
| ボタン | マグネティックホバー（useMagneticButton） | あり | なし |
| カーソル | ヒーローでカスタムカーソル | あり | なし |
| 数字 | ビューポート進入時のカウントアップ | あり | あり |
| ナビリンク | ホバー時アンダーライン拡張 | あり | 該当なし（モバイルメニュー） |
| アメニティ行 | ホバーで微かに拡張＋説明表示 | あり | タップで展開 |
| サービス行 | スクロールトリガーのリビール | あり | あり |
| ファウンダー経歴 | 行ごとの順次フェードイン | あり | あり |
| prefers-reduced-motion | すべてのアニメーション無効、コンテンツ即時表示 | あり | あり |

## API：お問い合わせフォーム

### POST /api/contact

**リクエストボディ：**
```typescript
{
  name: string       // 必須
  email: string      // 必須、バリデーションあり
  phone?: string
  category: string   // 必須："court" | "lesson" | "press" | "other"
  message: string    // 必須
}
```

**動作：**
- 必須フィールドとメールフォーマットのバリデーション
- Resend経由で hello@rstagency.com にメール送信
- 成功時 201、バリデーションエラー時 400、送信失敗時 500 を返却

## レスポンシブブレークポイント

- **デスクトップ：** 1440px — フル2カラムレイアウト、大きなタイポグラフィ、フルの余白
- **タブレット：** 768px — ほぼ1カラム、タイポグラフィ約70%スケール（ヒーロー56〜80px）、ハンバーガーメニュー
- **モバイル：** 375px — 完全1カラム、ヒーロー36〜48px、本文15〜16px、フルワイドボタン、タッチターゲット最低44px

## 画像プレースホルダー

実際の写真がまだ用意されていないため、画像エリアにはダークトーンの背景色と、意図する雰囲気と構図を説明するテキストを配置する。汎用的な「画像プレースホルダー」ボックスではなく、デザインの重量感と雰囲気を維持するスタイル。

## アクセシビリティ

- セマンティックHTMLを徹底（`div` + `onClick` は禁止）
- すべての画像に `alt` 属性
- すべてのフォーム入力に関連する `label`
- すべてのインタラクティブ要素にキーボードナビゲーション対応
- すべてのアニメーションで `prefers-reduced-motion` を尊重
- WCAG 2.1 AA カラーコントラスト準拠
- ARIAは控えめに使用、ネイティブHTMLセマンティクスを優先

## テスト

すべてのコンポーネントとフックはTDD（Red → Green → Refactor）に従う。テストはソースファイルと同じ場所に配置。100%カバレッジ目標を維持。
